# Тестирование Math-агента в офисе

После того как развёрнут `Workflow/math-agent.json` в n8n, привязаны credentials (Postgres + GigaChat) и активирован workflow — прогоняем эти кейсы и заполняем колонки «Получено» + «✓/✗».

Колонки `Ветка` — куда классификатор должен направить запрос:
- **PURE** — pure_expr (sympify без LLM, детерминированно)
- **MATH** — LLM генерирует Python с sympy
- **REEX** — reexplain (переиспользование code/raw_result)
- **GEN** — general chit-chat (без вычислений)
- **VAL** — отклонено валидацией

## Если что-то упало

1. **Открыть DevTools (F12) → Network**. Найти запрос на `/webhook/math` или `/webhook/math-explain`. Посмотреть **Response**:
   - `{response: "GigaChat-LLM недоступен"}` → LLM не запущен / неправильные creds
   - `{response: "Pyodide worker не инициализировался"}` → .whl файлы не на месте
   - `{response: "Запрос отклонён: ..."}` → валидация Phase 2 отказала
   - `{response: "Не удалось решить — Python завершился с ошибкой: ..."}` → Python exception в Pyodide
   - `{response: "Ответ: ошибка парсинга"}` → sympify не понял выражение (pure_expr ветка)

2. **DevTools → Console**: проверить ошибки от worker.

3. **n8n → Executions**: посмотреть выполнение workflow, какой узел упал.

4. Скрин Network + Console → присылай, разберём точечно.
