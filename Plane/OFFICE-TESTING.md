# Plane-агент — функциональное тестирование LLM

100 запросов от простого к сложному. Для каждого — ожидаемый результат.

**Тестовая среда**: проекты **«Общие задачи»** и **«Специальные задачи»**. В них несколько задач разных статусов (Отложено, Запланировано, В работе, Готово, Отменено). Sticky-project не задан (если иное не указано).

---

## А. Базовые приветствия и идентичность (1-5)

**1.** Запрос: `Привет`
Ожидаемо: `action: chat` — короткое представление «Я Plane-агент». БЕЗ слова «Привет» в начале.

**2.** Запрос: `Кто ты?`
Ожидаемо: `action: chat` — «Я Plane-агент» (краткое объяснение функций).

**3.** Запрос: `Меня зовут Георгий`
Ожидаемо: `action: chat` — «Запомнил, Георгий. Чем помочь?». НЕ «Меня зовут Георгий».

**4.** Запрос: `Как меня зовут?` (после теста 3)
Ожидаемо: `action: chat` — «Георгий» (из истории).

**5.** Запрос: `Как тебя зовут?`
Ожидаемо: `action: chat` — «Я Plane-агент» (НЕ имя юзера).

Ожидаемо: `action: create_issue, params: { project_name: "Общие задачи", name: "Подготовить отчёт", description: "Квартальный отчёт за Q2" }`.

Ожидаемо: `action: create_issue` — Plane создаст дубликат, это нормально (LLM не должен предотвращать).

**21.** Запрос: `Создай задачу с пустым именем` / `Создай задачу ""`

**22.** Запрос: `Создай задачу "Очень длинное название которое тянется и тянется на много много символов"`
Ожидаемо: `action: create_issue` со всем именем (Plane сам обрежет если превысит лимит).

**23.** Запрос: `Создай задачу в Общих с эмодзи 🚀 Запуск`
Ожидаемо: `action: create_issue, params: { project_name: "Общие задачи", name: "🚀 Запуск" }`.

**24.** Запрос: `Создай 3 задачи: А, Б, В в Общих задачах`
Ожидаемо: `action: bulk, params: { actions: [{create_issue A}, {create_issue B}, {create_issue C}] }`.

**25.** Запрос: `Сделай задачу "Тест" в проекте общие задачи` (нижний регистр)
Ожидаемо: case-insensitive matching → `create_issue, params: { project_name: "Общие задачи", name: "Тест" }`.

## E. Smart-prioritization и smart-dates (26-33)

**26.** Запрос: `Срочно: упал сайт, нужен фикс`
Ожидаемо: `action: create_issue, params: { ..., priority: "urgent" }` (по маркерам «срочно», «упал»).

**27.** Запрос: `Создай задачу с высоким приоритетом "Документация"`
Ожидаемо: `action: create_issue, params: { ..., priority: "high" }`.

**28.** Запрос: `Когда будет время, добавь мелочь в дизайн`
Ожидаемо: `action: create_issue, params: { ..., priority: "low" }` (по маркерам «когда будет время», «мелочь»).

**29.** Запрос: `Создай "Обзор кода" со сроком до пятницы`
Ожидаемо: `action: create_issue, params: { ..., target_date: "<ближайшая пятница YYYY-MM-DD>" }`. Расчёт от текущей даты в шапке промпта.

**30.** Запрос: `Задача "Подготовка митапа" к концу месяца`
Ожидаемо: `action: create_issue, params: { ..., target_date: "<последний день текущего месяца>" }`.

**31.** Запрос: `Создай задачу "Релиз" через 2 недели`
Ожидаемо: `target_date = today + 14 дней`.

**32.** Запрос: `Установи срок задачи "Купить хлеб" на завтра`
Ожидаемо: `action: set_deadline, params: { ..., target_date: "<today + 1 день>" }`.

**33.** Запрос: `Поставь срок до 25 декабря`
Ожидаемо: `target_date: "YYYY-12-25"` где YYYY — текущий год или следующий, если 25 декабря уже прошло.

## Как использовать

1. Запускай каждый запрос в чате Plane-агента.
2. Для каждого фиксируй: ✅ если результат совпал, ❌ если нет.
3. Для ❌ записывай: что ожидал → что получил.
4. Отдай мне список багов — я внесу правки в системный промпт и/или код.

**Критичные провалы** (нужны фиксы):
- Английские слова в ответе
- Markdown / ```json блоки вместо чистого JSON
- `action: chat` там где должен быть реальный action (create/update/...)
- «Меня зовут <имя_юзера>» в идентичности
- Выдуманные проекты/задачи

**Допустимые расхождения** (не критично):
- Слегка другая формулировка `text` поля
- `chat` с просьбой уточнить вместо догадки (если контекст реально неоднозначный)
- Mine + overdue → приоритет любой из двух filter'ов
