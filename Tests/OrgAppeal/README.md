## Как использовать

1. Открой **Дашборд GigaChat → Алгоритмы → Организация обращения**
2. Перетащи `.docx` в зону загрузки или выбери кнопкой
3. Нажми «Обработать обращение»
4. Сверь итог с колонкой «Что проверяем»

Все 7 файлов нужно прогнать, чтобы убедиться: workflow корректно ловит все 7 веток.

## Перегенерация

Если нужно изменить шаблон или добавить кейсы — отредактируй `generate.py` и запусти:

```
python generate.py
```

Скрипт использует только стандартную библиотеку Python (`zipfile`, `xml`), внешних пакетов не требует.

## Структура .docx

Каждый файл — минимально валидный OOXML-документ (3 файла внутри ZIP):

- `[Content_Types].xml`
- `_rels/.rels`
- `word/document.xml`

Текст хранится в UTF-8 в `<w:t>` элементах. Открывается MS Word, LibreOffice, и любыми OCR/text-extraction утилитами (включая наш `http://localhost:8080/v1/file/text/`).
