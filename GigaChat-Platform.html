Шлюз API для внешних проектов

Базовый URL шлюза

—

Используется внешними проектами как корневой URL. Все запросы идут на <base>/webhook/<agent>.

Доступные агенты

/webhook/routerАвто-маршрутизация — определит нужного агента
/webhook/chat-agentУниверсальный чат-агент
/webhook/rag-agentПоиск ответов в загруженных документах
/webhook/sql-agentЗапросы к базе данных на естественном языке
/webhook/math-agentМатематические вычисления, Python-код
/webhook/prompt-engineerГенерация и улучшение промптов

Формат запроса

POST <base>/webhook/<agent> Content-Type: application/json { "message": "Текст вопроса", "session_id": "myproject_user42_chat" }

Формат ответа

{ "response": "Ответ агента" }

Изоляция сессий — ВАЖНО

Каждый внешний проект должен добавлять свой префикс к session_id, иначе сессии пересекутся с GigaChat HTML:

  • myproject_user42_chat — внешний проект «myproject»
  • otherapp_admin_rag — другой проект
  • chat_1747234567_abc123 — формат GigaChat HTML, не использовать

Готовый Django app

Для соседних проектов на Django — в репозитории есть готовая папка Agents/django-example/. Скопируйте её в Django-проект как app giga, добавьте в INSTALLED_APPS и подключите path('giga/', include('giga.urls')). Получите endpoint POST /giga/ask для вызова любого из 6 агентов из вашего HTML. Работает на Django 3.x / 4.x / 5.x.

Полная инструкция

В корне репозитория лежит INTEGRATION.md — описывает все варианты подключения, обработку ошибок, изоляцию сессий и развёртывание.

Отправь тестовый запрос напрямую в шлюз — увидишь ответ как его получит внешний проект.
