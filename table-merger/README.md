# Table Merger — сервис объединения таблиц

Локальный HTTP-сервис, который принимает несколько файлов (Excel `.xlsx` или Word `.docx`), извлекает из каждого одну таблицу и склеивает их в один итоговый файл того же формата. Используется через дашборд GigaChat → раздел «Инструменты» → «Объединение таблиц».

## Что делает

- На входе **N файлов одного формата** (Excel или Word, не смешивать).
- В **Excel** читается **первый лист**, первая строка считается заголовком.
- В **Word** читается **первая таблица** в документе, первая строка таблицы — заголовок.
- Заголовки столбцов из всех файлов **объединяются в один порядок**: первые столбцы первого файла → лишние из второго → и т.д.
- Строки склеиваются вертикально. Если в каком-то файле не было определённого столбца — соответствующая ячейка в итоге пустая.
- На выходе **один файл того же формата** с объединённой таблицей.

## Как сопоставляются столбцы

Заголовки сопоставляются по «нормализованному» виду — без пробелов, точек, скобок, дефисов, регистра:

| Оригинал | Нормализованный |
|---|---|
| `Ф.И.О.` | `фио` |
| `Ф И О` | `фио` |
| `ФИО` | `фио` |
| `(Сумма)` | `сумма` |
| `Сумма, ₽` | `сумма₽` |

Это закрывает 90% реальных кейсов «таблицы одинаковые, но кто-то лишний пробел поставил». Если же столбцы реально разные — они оба попадут в итог как отдельные.

В итоговом файле **отображаемое имя** столбца берётся из **первого встреченного** оригинала (стабильный порядок).

## Установка на офисном ПК (офлайн)

> Этот сервис ставится **отдельно от папки проекта GigaChat** — в `C:\table-merger\`. Это сделано чтобы обновления ZIP-проекта не ломали виртуальное окружение venv.

**1. Подготовка на ДОМАШНЕМ ПК (где есть интернет):**

```powershell
cd C:\Users\Lenovo\Desktop\GigaChat\table-merger
.\make-offline-bundle.ps1
```

Скрипт скачает все Python-пакеты в `wheels.zip` (~15 МБ). После этого скопируй на флешку папку `table-merger` (с `wheels.zip` внутри).

**2. На ОФИСНОМ ПК:**

```
C:\table-merger\
├── server.py
├── requirements.txt
├── install-offline.ps1
├── start.bat
└── wheels.zip
```

Скопируй эти файлы из флешки в `C:\table-merger\` (создай папку если её нет).

Открой PowerShell в `C:\table-merger\` и запусти:

```powershell
.\install-offline.ps1
```

Скрипт:
- Распакует `wheels.zip` в подпапку `wheels\`.
- Создаст виртуальное окружение `venv\`.
- Установит все пакеты из `wheels\` **без интернета**.

После «УСТАНОВКА УСПЕШНА» сервис готов к запуску.

## Запуск

**Вариант 1** — двойной клик по `start.bat`. Откроется консольное окно с логами. Закрытие окна = остановка сервиса.

**Вариант 2** — из PowerShell:
```powershell
cd C:\table-merger
.\venv\Scripts\Activate.ps1
python server.py
```

Проверка что работает:
```powershell
curl http://localhost:8082/health
```
Должно вернуть `{"status":"ok",...}`.

## API

### `POST /merge`

Принимает `multipart/form-data` с полем `files[]` (несколько файлов).

- Все файлы должны быть **одного формата** (все `.xlsx` или все `.docx`).
- Жёсткий предел — 25 файлов (настраивается через `MERGER_MAX_FILES`).
- Возвращает binary итогового файла с правильным `Content-Type` и `Content-Disposition`.

Пример с curl:
```powershell
curl.exe -X POST http://localhost:8082/merge `
  -F "files=@C:\data\report1.xlsx" `
  -F "files=@C:\data\report2.xlsx" `
  -o merged.xlsx
```

### `GET /health`

```json
{"status":"ok","port":8082,"max_files":25}
```

## Ограничения

- **Только первый лист в Excel** — если данные на втором листе, переложи в первый перед загрузкой.
- **Только первая таблица в Word** — если в документе несколько таблиц, остальные игнорируются.
- **Первая строка = заголовок** — если в файле есть «шапка» (название, дата, автор) над таблицей, удали её перед загрузкой.
- **Без слияний ячеек в заголовках** — если заголовок объединён через несколько столбцов, результат может быть неточным.
- **Большие файлы** — 50K+ строк работают, но медленно (полминуты на файл). Для регулярных задач такого объёма лучше использовать pandas-скрипт.

## Обновления

Этот сервис **полностью изолирован** от папки проекта `GigaChat`:
- Папка `C:\table-merger\` существует отдельно.
- В ней — `venv\`, `wheels\`, `wheels.zip`, `server.py`.
- Обновление проекта (новый ZIP с GitHub) НЕ затрагивает эту папку.

Когда сам сервис обновлён в репозитории (изменился `server.py` или зависимости в `requirements.txt`):
1. Скопируй новые файлы из проекта в `C:\table-merger\`.
2. Если поменялся `requirements.txt` — пересобери бандл (`make-offline-bundle.ps1` дома → принеси новый `wheels.zip` → перезапусти `install-offline.ps1`).
3. Перезапусти сервис.

Если поменялся только `server.py` — достаточно скопировать его и перезапустить, venv остаётся.
